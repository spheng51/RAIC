# Execution Plan: v0.6.0 Adaptive Student Beta Readiness

Date: 2026-05-18

## Goal

Prepare Open-RAIC for a reviewed Adaptive Student Beta without enabling student-facing adaptation by default. The release must add a reversible server-side gate, document privacy/consent/retention/API requirements, and prove teacher-only adaptive behavior cannot leak into student, public demo, anonymous, or classroom-cookie paths.

## Current Baseline

- `v0.5.0` is tagged and released from `main`.
- Teacher-only adaptive context is available through session context, reflection, generation, chat, scene regeneration, and PBL chat paths.
- Public/student request and response payloads remain unchanged.
- Issues are disabled, so roadmap docs, execution plans, release evidence, and PR descriptions remain the source of truth.

## Implementation Contract

- Add `RAIC_STUDENT_ADAPTATION_BETA`; default unset/`false`.
- The flag alone must not activate student adaptation. Student prompt reuse requires a later reviewed consent and retention implementation.
- Keep teacher/internal adaptive context restricted to authenticated teacher web access.
- Do not add public/student API fields in the first readiness slice.
- Rollback is immediate: set `RAIC_STUDENT_ADAPTATION_BETA=false` or unset it and redeploy.

## Privacy, Consent, Retention, And API Review

- Privacy: student-facing adaptation may only use classroom-scoped learning signals and must exclude teacher/internal analytics.
- Consent: no student signal may be reused in prompts until an explicit classroom-scoped consent mechanism exists.
- Retention: any future student beta records must be covered by platform retention cleanup before release.
- Public API: first readiness slice adds only an operator env flag. Any later request/response shape change needs a dedicated PR and updated release evidence.

## Backlog Disposition

- Defer Discord integration and scheduled reminder work to a separate integration milestone.
- Compare duplicate `* 2.*` files against tracked originals before archiving or deleting them.
- Treat the schedule empty-state change as possible `v0.5.1` UX hardening, not v0.6.0 readiness.
- Defer demo video and game-arcade sequence work.

## Test And Release Gates

- Add non-leakage tests for student, classroom-cookie, public demo, anonymous, and teacher paths.
- Add rollback tests proving the readiness flag does not grant student adaptive context without reviewed consent.
- Preserve teacher adaptive prompt tests for generation, chat, scene regeneration, and PBL chat.
- Run focused tests, then `corepack pnpm test`, `corepack pnpm exec tsc --noEmit`, `corepack pnpm run check:i18n-keys`, and the repo release/ops gates under Node 24 before tagging.

Branch readiness evidence is tracked in [Release Evidence: v0.6.0 Adaptive Student Beta Readiness](../release-evidence-v0.6.0.md). Final `secrets:scan`, `ops:drift`, benchmark, `ops:verify`, and production smoke evidence must still be captured on clean local `main` before tagging.
